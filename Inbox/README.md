# Входящие материалы

[← Точка входа](../README.md) | [Задачи](../Issues/README.md) | [Реестр задач](../Issues/issue_registry.jsonl) | [События](../Issues/issue_events.jsonl)

`Inbox/` — временное хранилище входных материалов до их привязки к задаче, переноса в рабочее совместимое место или очистки. Это зона подготовки, а не кладбище файлов, хотя человечество явно старалось.

## Управление

| Поле | Значение |
|---|---|
| Владелец | `Service Mode` |
| Источник истины | `Issues/README.md`, `Issues/issue_registry.jsonl`, `Issues/issue_events.jsonl` |
| Глобальный маршрут | `README.md` → `Inbox/README.md` |
| Разрешённое содержимое | только входящие материалы с последующей классификацией |

## Правила

1. Входные материалы получают связь с задачей до ответа агента.
2. Материалы не становятся частью готовой концепции автоматически.
3. После закрытия связанных задач выполняется очистка или перенос с сохранением происхождения.
4. Рабочие контрольные архивы реализации сюда не помещаются.
5. Разработческие материалы исправления Phase 2, аудит, prompt-ы и исходный архив передачи не загружаются в рабочий репозиторий.
