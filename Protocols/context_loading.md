# Загрузка контекста

[← Реестр протоколов](protocol_index.md) | [Архитектура состояния](state_architecture.md)

## Назначение

Протокол задаёт порядок чтения файлов перед действием агента. Цель простая: брать достаточно контекста для точного шага и не тащить весь репозиторий в каждый ответ.

## Базовый порядок

1. Прочитать `README.md`.
2. Прочитать state активного режима.
3. Прочитать `Protocols/protocol_index.md`.
4. Выбрать только протоколы, нужные для текущего действия.
5. Прочитать связанные реестры, если действие меняет файлы, state или `issue`.
6. Не читать файлы конкретной концепции без выбранного `active_object`.

## Контекст Service Mode

Минимальный набор:

```text
README.md
State/service_state.md
Protocols/protocol_index.md
```

Дополнительно загружаются только файлы из `Instructions/`, `Protocols/`, `Issues/`, `Registry/` или `Inbox/`, если они нужны для текущей задачи.

## Контекст Execution Mode

Минимальный набор:

```text
README.md
State/execution_state.md
Concepts/README.md
Protocols/protocol_index.md
```

Файлы концепции читаются только после выбора концепции и проверки области записи.

## Ограничение объёма

Если файл не нужен для решения текущей задачи, он не загружается. Если без него нельзя проверить запись, файл загружается после записи для контрольного чтения.

## Ошибка загрузки

Если нужный файл отсутствует, агент фиксирует блокировку, не подменяет файл догадкой и создаёт сервисное `issue`, если это разрешено текущим режимом.
