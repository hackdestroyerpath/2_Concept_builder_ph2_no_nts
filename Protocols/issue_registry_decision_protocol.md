# Протокол решений по реестру `issue`

[← К реестру протоколов](protocol_index.md) | [Issues](../Issues/README.md)

## Цель

Обрабатывать решения пользователя по кандидатам `issue`: утвердить, отклонить, обсудить, добавить или отложить.

## Допустимые команды

| Команда | Действие |
|---|---|
| `утвердить всё` | все `proposed` переходят в `approved` |
| `утвердить ISSUE-ID` | один кандидат переходит в `approved` |
| `отклонить ISSUE-ID` | кандидат переходит в `rejected` с reason |
| `обсудить ISSUE-ID` | открывается фокус на одном кандидате |
| `добавить issue: ... reason: ...` | создаётся новый кандидат с обязательным reason |
| `отложить ISSUE-ID` | кандидат получает статус `needs_discussion` или `blocked` |

## Процесс

1. Загрузить registry, events и reason выбранных кандидатов.
2. Проверить, что команда относится к текущему `active_menu_id`.
3. Обновить статусы в registry.
4. Для каждого изменения записать event.
5. Если кандидат утверждён, определить следующий stage: QA, requirements или dependency check.
6. Если кандидат отклонён, сохранить provenance и применить cleanup policy.
7. Перечитать registry и events.
8. Вернуть пользователю таблицу изменённых `issue` и следующий безопасный шаг.

## Выход

Обновлённый registry, события и marker с новым `dialogue_state`.
