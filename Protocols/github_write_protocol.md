# Протокол записи в GitHub

[← Реестр протоколов](protocol_index.md) | [Загрузка контекста](context_loading.md) | [Маркер ответа](response_marker.md)

## Назначение

Протокол описывает безопасную запись production-файлов в репозиторий `GitHub`.

## Перед записью

Агент фиксирует:

```text
mode:
active_object:
active_issue:
reason:
target_paths:
related_registries:
expected_state_change:
validation_plan:
```

## Разрешённые действия

1. Создать новый production-файл, если путь входит в область записи активного режима.
2. Обновить существующий production-файл после чтения его текущего `sha`.
3. Обновить связанные реестры и state в том же ограниченном этапе.
4. Сохранить рабочие материалы вне production-репозитория.

## После записи

1. Перечитать каждый изменённый файл из `GitHub`.
2. Проверить, что ключевые строки присутствуют.
3. Проверить, что ссылки указывают на существующие production-файлы или явно планируемые файлы.
4. Обновить `persistence_status`.
5. Вернуть sync-report пользователю.

## Статусы persistence

| Статус | Значение |
|---|---|
| `not_written` | Запись не выполнялась. |
| `written_unverified` | Запись выполнена, но проверочное чтение ещё не прошло. |
| `synced` | Запись выполнена и перечитана из `GitHub`. |
| `blocked` | Запись остановлена с причиной. |

## Ограничения

В production-репозиторий не загружаются checkpoint-архивы, стартовые задания, методология реализации, промежуточные отчёты и временные рабочие материалы.
