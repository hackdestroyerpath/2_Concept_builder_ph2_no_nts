# Начальная загрузка Execution Mode

[← Реестр протоколов](protocol_index.md) | [Состояние исполнения](../State/execution_state.md) | [Концепции](../Concepts/README.md) | [Экспорт концепции](concept_export.md)

## Назначение

`execution_bootstrap` описывает безопасный вход в `Execution Mode`: выбор существующей концепции или создание новой концепции из рабочего шаблона. Этот протокол не меняет файлы сервисного управления; он только выбирает область концепции и подготавливает следующий исполнительный маршрут.

## Когда применяется

`execution_bootstrap` применяется, когда пользователь просит создать, выбрать или продолжить конкретную концепцию внутри `Concepts/`.

## Входные данные

```text
command:
concept_id:
concept_title:
mode:
active_issue:
source_materials:
acceptance_criteria:
```

`concept_id` должен быть коротким техническим идентификатором без пробелов. Название концепции хранится отдельно как `concept_title`.

## Обязательный контекст

| Контекст | Назначение | Условие блокировки |
|---|---|---|
| `README.md` | корневой маршрут и выбор режима | корневой маршрут отсутствует |
| `State/execution_state.md` | активный исполнительный объект, области и поля маркера | состояние отсутствует или неканоническое |
| `Concepts/README.md` | перечень концепций и политика проверочного примера `smoke` | выбранная концепция недостижима |
| `Protocols/protocol_index.md` | матрица маршрутов и следующий протокол | карта маршрута отсутствует |
| `Protocols/mode_routing.md` | проверка границы `Service Mode` / `Execution Mode` | смешанная область не разрешена |
| `Templates/concept/README.md` | шаблон создания новой концепции | маршрут шаблона отсутствует для действия создания |
| `Concepts/<concept_id>/page_registry.jsonl` | локальный реестр существующей концепции | отсутствует локальный реестр для продолжения |

## Матрица маршрутов

| Сценарий | Предусловия | Выход | Следующий протокол |
|---|---|---|---|
| Выбрать существующую концепцию | папка есть, локальный реестр есть, активная задача известна или не требуется для чтения | `active_object=Concepts/<concept_id>` и ограниченная область чтения/записи | `concept_export` или работа конкретной концепции |
| Создать новую концепцию | валидный `concept_id`, есть задача/происхождение, маршрут шаблона существует, целевая папка отсутствует | план папки новой концепции и пакет записи | `github_write_protocol` |
| Продолжить задачу концепции | строка связанной задачи и трасса событий существуют | продолженная область концепции и якорь возврата | целевой протокол задачи концепции |
| Проверить пример `smoke` | `Concepts/smoke` зарегистрирован как проверочный пример | доказательства примера только для чтения | `validation_protocol` или ремонт `P2-009` |
| Смешанный сервисный/исполнительный запрос | сервисная цель и цель концепции присутствуют одновременно | маршрут передачи или заблокированное разделение | `mode_routing` |

## Создание концепции

1. Проверить, что задача относится к `Execution Mode`.
2. Создать или подтвердить задачу на создание концепции.
3. Проверить `concept_id`, чтобы не перезаписать существующую папку.
4. Скопировать структуру из `Templates/concept/` в `Concepts/<concept_id>/`.
5. Заполнить `README.md`, `structure.md`, `page_registry.jsonl` и стартовые страницы концепции.
6. Обновить `State/execution_state.md`, глобальный `Registry/page_registry.jsonl` и журналы `Issues/`.
7. Запустить `Protocols/validation_protocol.md`.

## Выбор существующей концепции

1. Найти папку `Concepts/<concept_id>/`.
2. Прочитать `Concepts/<concept_id>/README.md` и локальный `page_registry.jsonl`.
3. Проверить обратные ссылки, маршрут результата/экспорта и связанную задачу, если запись планируется.
4. Обновить `State/execution_state.md` только в рамках разрешённой области `Execution Mode` или утверждённого сервисного ремонта.
5. Вернуть маркер ответа с выбранным `active_object`.

## Карта протокола

| Поле | Значение |
|---|---|
| Владелец | `Execution Mode` |
| Триггер | запрос создать, выбрать или продолжить концепцию; проверка `smoke`; запрос готовности экспорта |
| Входы | команда, `concept_id`, `concept_title`, активная задача, исходные материалы, критерии приёмки |
| Обязательный контекст | состояние исполнения, точка входа концепций, индекс протоколов, маршрутизация режимов, локальный реестр или шаблон концепции |
| Выходы | область выбранной концепции, план создания, причина блокировки или следующий исполнительный протокол |
| Область записи | выбранная `Concepts/<concept_id>/` и связанная исполнительная задача; сервисные файлы только через передачу |
| Следующий протокол | `concept_export`, протокол задачи концепции, `github_write_protocol` или `validation_protocol` |
| Условия блокировки | отсутствует `concept_id`, отсутствует локальный реестр, конфликт целевой папки, нет задачи для записи, смешанная сервисная область |

## Блокировки

Концепция не создаётся и не выбирается для записи, если отсутствует `concept_id`, неясен режим, конфликтует путь, нет задачи для записи или нарушена область записи. При конфликте области агент фиксирует маршрут передачи, а не пишет сервисные и исполнительные файлы одним пакетом.

## Проверочные заметки P2-004

P2-004 связывает `execution_bootstrap` с глобальным индексом протоколов, загрузкой контекста, маршрутизацией режимов и экспортом концепции. Детальный пробный прогон `smoke`/экспорта остаётся задачей P2-009/P2-010.
