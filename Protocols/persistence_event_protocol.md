# Протокол событий сохранения

[← К реестру протоколов](protocol_index.md) | [События issue](../Issues/issue_events.jsonl)

## Цель

Единообразно фиксировать события, которые меняют state, registry, issue или production-артефакты.

## Когда применять

- создан или обновлён `issue`;
- изменён статус lifecycle;
- сохранены QA, requirements, solution, contract, output или report;
- добавлена зависимость между `issue`;
- выполнена запись production-файла;
- обнаружен конфликт, частичная запись или ошибка проверки.

## Минимальная запись события

```json
{"event_id":"evt-...","event_type":"...","issue_id":null,"mode":"Service Mode","status":"synced","paths":[],"reason":"...","created_at":"YYYY-MM-DDTHH:MM:SSZ"}
```

## Правила

1. Событие создаётся до ответа пользователю, если ответ сообщает о сохранённом изменении.
2. `paths` содержит только production-файлы, реально затронутые событием.
3. `reason` кратко объясняет, почему переход был выполнен.
4. При частичной записи событие помечается `status: partial` и указывает следующий шаг проверки.
5. События являются журналом, а не заменой state текущего объекта.
