# Протокол загрузки контекста

[← К реестру протоколов](protocol_index.md)

## Цель

Дать агенту ровно тот контекст, который нужен для текущего действия, без чтения всего репозитория.

## Уровни

| Уровень | Состав |
|---|---|
| `mandatory` | state текущего режима, входная страница, active issue, active protocol, return anchor |
| `local` | файлы текущего issue, концепции или стадии |
| `referenced` | конкретные файлы по ссылкам из state, registry, contract или dependency |
| `optional` | дополнительный контекст с явно указанной пользой |
| `excluded` | весь репозиторий, чужой режим, закрытая история без ссылки, рабочие архивы реализации |

## Порядок

1. Определить режим, объект, issue, стадию и return anchor.
2. Загрузить `mandatory`.
3. Добавить локальный слой текущего объекта.
4. Добавить только явно referenced файлы.
5. Зафиксировать `loaded_context` и причину достаточности в marker.
6. После закрытия шага свернуть детали в summary и вернуться к `return_anchor`.

## Критерий достаточности

Контекста достаточно, если агент может назвать режим, объект, цель шага, источник истины, нужный протокол, ожидаемый результат и файл возврата.
